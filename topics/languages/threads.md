

| Term             | Where in stack                                 | What it really is                                                                                                                                                                                                               |
| ---------------- | ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `pthread`        | OS/API boundary                                | POSIX C API for creating and managing threads in a process; threads share process memory but each has its own stack [man](https://man.cx/pthreads\(7\)).                                                                        |
| platform thread  | Language runtime over OS thread                | In Java, a platform thread is a thin wrapper around an OS thread and captures that OS thread for its whole lifetime [dev](https://dev.to/birajdar/platform-thread-vs-virtual-thread-in-java-16n7).                              |
| `std::thread`    | C++ standard library over OS/native thread API | C++ object representing one thread of execution, typically backed by the platform’s native thread facilities [cppreference](https://en.cppreference.com/w/cpp/thread/thread.html).                                              |
| carrier thread   | JVM runtime internals                          | A platform thread currently carrying a mounted virtual thread; it is not a third kind of thread, just a role a platform thread plays dev+1.                                                                                     |
| green thread     | Runtime/user-space scheduling concept          | A user-space managed thread-like unit scheduled by a language runtime rather than directly by the OS [kushallabs](https://kushallabs.com/understanding-concurrency-in-go-green-threads-os-threads-and-goroutines-b40a4ec14981). |
| virtual thread   | JVM runtime abstraction                        | A Java `Thread` implementation managed by the Java runtime, mapped onto a smaller number of OS/platform threads [dev](https://dev.to/birajdar/platform-thread-vs-virtual-thread-in-java-16n7).                                  |
| goroutine        | Go runtime abstraction                         | Go’s lightweight user-space concurrency unit, multiplexed by the Go runtime over OS threads [kushallabs](https://kushallabs.com/understanding-concurrency-in-go-green-threads-os-threads-and-goroutines-b40a4ec14981).          |
| Kotlin coroutine | Language/library async abstraction             | A suspendable computation managed by Kotlin and `kotlinx.coroutines`, executed on dispatchers that use threads or thread pools underneath stackoverflow+1.                                                                      |
